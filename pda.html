DPDA Example — Palindrome WCWR

Language Definition

The language consists of all strings that are of the form W c WR, where W is any string of {a, b}* and WR represents the reverse of W.

L = { W c WR | W ∈ {a, b}* }

Formal Definition

The Deterministic Pushdown Automata (DPDA) for this language is defined as:

M = ( Q, Σ, Γ, δ, q0, Z0, F )

Where:
Q = { q0, q1, q2, q3 }
Σ = { a, b, c }
Γ = { a, b, Z0 }
Start state: q0
Initial stack symbol: Z0
Final state: q3

Conceptual Description

This DPDA checks if a given string follows the palindrome pattern separated by a center symbol ‘c’. It works in three main phases:

  • Push Phase (q₁): The automaton reads W and pushes each symbol onto the stack.
  • Pivot Phase (q₂): Upon reading the center marker ‘c’, it switches from push mode to match mode.
  • Match Phase (q₂ → q₃): For every symbol in WR, the automaton pops and matches the stack top. If the stack is empty when input ends, the string is accepted.

Transition Rules

1. Initialize Stack:
( q0, ε, ε ) → ( q1, Z0 )

2. Push Phase:
( q1, a, ε ) → ( q1, a )
( q1, a, a ) → ( q1, aa )
( q1, a, b ) → ( q1, ab )
( q1, b, a ) → ( q1, ba )
( q1, b, b ) → ( q1, bb )

3. Pivot Phase:
( q1, c, a ) → ( q2, a )
( q1, c, b ) → ( q2, b )

4. Match Phase:
( q2, a, a ) → ( q2, ε )
( q2, b, b ) → ( q2, ε )

5. Accept:
( q2, ε, Z0 ) → ( q3, ε )

Conclusion

Hence, the DPDA successfully recognizes all strings of the form W c WR. It uses the stack to store the first half and then verifies its mirror structure after the center symbol.
